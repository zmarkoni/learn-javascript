jQuery parallax slider plugin

FractionSlider 0.9.9.9

FractionSlider is a jQuery plugin for image/text-sliders.

It allows you to animate multiple elements per slide. You can set different animation methods like fade or transitions from a certain direction. Also there are options to specify delays and easing for each element. You have full layout and design control through html and css (every html-element can be animated).


New version: 0.9.9.9 IE bugfix: images without width/height get calculated correctly

pager: You can now set a jQuery-Object as pager-wrapper outside of the slider

New version: 0.9.9.8 New features arriving:

Plugin option: pauseOnHover
The slider has now the pause on hover functionality.
Plugin option: increase
The slider can get bigger than the specified dimensions (useful for responsive design)
Plugin options: callbacks
I added support for a lot of custom callback functions (see Plugin options for more information)

New version: 0.9.9.4 Finally some IE bugfixing. Let me know how it works in ie!

New version: 0.9.9 Some minor bugfixing

New version: 0.9.8 Plugin methods (pause, resume, stop…) are available – Learn how to use them

New version: 0.9.5 Introduction of slide transitions Learn how to use them or look at the Demo

IE Bugs Some people told me, that there are problems with IE 8,9 and 10, but i couldn´t reproduce them – please let me know if you have any problems with IE!

New: FractionSlider is now responsive (but still beta!). Learn how to activate it.

Attention: Since 0.8.0 the data-speed attribute is replaced by data-time. Please see the Element options reference for further information.

  62. Gary Francis
    19. Juli 2013 at 23:26

    For those of you who are having some problems in IE, here’s something I discovered that might help you out.

    If you have your slide working as expected in Chrome and other browsers, but in Internet Explorer (all versions including 10), your images seem to disappear then there’s a simple fix.

    You need to set the height and width of your image as a CSS style (e.g. style=”width: 100px; height: 100px”). The Javascript that calculates the width is unable to work out the size if you don’t specifically set it as a CSS style, whereas other browsers such as Chrome use the default size of the image.

    I’m pretty sure this only applies to slides that are using the “resposive: true” property, but I haven’t done too much further testing.

    Gary
